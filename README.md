○この暗号について
  入力された文字の他に暗号化強度の分のダミーを追加しています。
  素因数分解をして、ダミーを排除してからその素数が何番目かを求めることで答えを求めることができます。

○ダミーの排除のやり方
  暗号を素因数分解すると、ダミーの素数は偶数個出てきますが、本物の素数は奇数個出てきます。
    (例) 440657 => 23 * 23 * 17 * 7 * 7
  この例だと17が奇数個出てきているのでダミーは23と7です。
  よってこの関数でのリターンは17となります。

○何番目かを求める
  ここで、素数が何番目かを求める関数を使います。
    (例) 1 => 2, 2 => 3, 3 => 5, 4 => 7, 5 => 11
  これによって素数ではない数字でも暗号化することができるようになっています。

○所要時間について
  ダミーは暗号化強度が増えるほど増えます。よって解読に時間がかかるようになりますが、暗号化にwも時間がかかります。
  暗号化はこれでもがんばったほうです。最適化前と比べて100倍くらい早くなってます。ほめて
